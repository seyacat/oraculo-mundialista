# 11 — UX/UI, Neuromarketing y Retención Frontend Review

## Veredicto ejecutivo

El frontend actual tiene una base visual correcta: dirección Stadium Social, fondo oscuro, acentos energéticos, cards deportivas y una narrativa de Oráculo. Sin embargo, todavía está en una fase inicial: se parece más a una demo de predicción/dashboard que a una red de comunidades virales.

Para cumplir el objetivo de retención y viralidad, el frontend debe reforzar cuatro loops:

1. Loop de comunidad: crear → invitar → progreso → beneficio.
2. Loop de competencia: predecir → ranking → subir/bajar → compartir.
3. Loop de reputación: gloria → certificado → vergüenza → revancha.
4. Loop de retorno: partido próximo → cierre cercano → miedo a quedar como Fantasma.

## Evaluación UX actual

### Lo que sí funciona

- El estilo visual es coherente con fútbol y competencia.
- El naming El Oráculo Mundial tiene fuerza.
- Los poderes agregan estrategia y diferenciación.
- La Tabla de la Vergüenza tiene potencial viral.
- El fondo oscuro + acento energético crea atmósfera de estadio.
- Ya existe una base PWA adecuada para mobile.

### Lo que falta

- WhatsApp debe ser más protagonista.
- La comunidad debe ser el centro, no el dashboard.
- La meta de jugadores debe estar visible todo el tiempo.
- El beneficio desbloqueable debe tener más peso emocional.
- El onboarding debe explicar el juego en menos de 5 segundos.
- La tabla de vergüenza debe tener momentos compartibles.
- El ranking debe mostrar comparación social clara.
- El usuario debe saber qué hacer después de cada acción.

## Principios neurolingüísticos aplicados

### 1. Identidad social

El copy debe hablar de “tu gente”, “tu comunidad”, “tu hinchada”, “tu grupo”.

Usar:

```txt
Lo juegas contra tu gente.
Reta a tu grupo.
Tu comunidad está creciendo.
Tu hinchada espera tu predicción.
```

No usar solo:

```txt
Enviar predicción.
Crear cuenta.
Ver dashboard.
```

### 2. Rivalidad segura

La rivalidad debe sentirse divertida, no agresiva.

Usar:

```txt
A ver si sabes tanto de fútbol como dices.
Se aceptan excusas y audios de 3 minutos.
Pide revancha.
Hoy el fútbol fue cruel con estos profetas.
```

### 3. Aversión a la pérdida social

El usuario debe sentir que no participar lo deja fuera de la conversación.

Usar:

```txt
No te quedes como El Fantasma de la jornada.
Predice antes de que cierre.
Tu comunidad ya está jugando.
Faltan 7 jugadores para desbloquear el beneficio.
```

### 4. Progreso visible

El cerebro responde bien a metas visibles y cercanas.

Mostrar siempre:

```txt
18/25 jugadores
72% completado
Faltan 7 para desbloquear el primer beneficio
```

### 5. Recompensa variable sana

No todos los partidos deben sentirse iguales. Usar:

- Sorpresa del partido.
- Certificado de Oráculo.
- Recibo de Realidad.
- Tabla de la Vergüenza.
- Poder desbloqueado.
- Beneficio comunitario.

No usar mecánicas de azar monetario.

## Evaluación por pantalla esperada

### Landing

Debe activar comprensión y deseo.

Debe responder:

- ¿Qué es?
- ¿Por qué me interesa?
- ¿Con quién juego?
- ¿Qué gano socialmente?
- ¿Cómo invito?

CTA principal: Crear mi comunidad gratis.

### Crear comunidad

Debe activar compromiso.

Pregunta crítica:

```txt
¿Cuántos jugadores quieres retar?
```

Esto convierte el formulario en una promesa social.

### Comunidad

Debe ser el centro del producto.

Debe mostrar:

- Progreso.
- Próximo partido.
- Ranking resumido.
- Próximo beneficio.
- CTA WhatsApp.

### Predicciones

Debe ser rápida.

No más de 10 segundos por predicción.

Estados visuales:

- Abierto.
- Cierra pronto.
- Cerrado.
- Guardado.

### Ranking

Debe mostrar jerarquía y movimiento.

Incluir:

- Top 3.
- Mi posición.
- Subió/bajó.
- Compartir posición.

### Vergüenza

Debe ser memeable.

Debe incluir:

- Categoría.
- Razón.
- Revancha.
- Compartir por WhatsApp.

### Poderes

Deben verse como recursos valiosos, no como decoración.

Cada poder debe responder:

- Qué hace.
- Cuándo usarlo.
- Cómo ganarlo.
- Cuántos usos quedan.

## Reglas de retención frontend

1. Después de crear comunidad → invitar por WhatsApp.
2. Después de unirse → predecir primer partido.
3. Después de predecir → ver ranking o invitar.
4. Después de ranking → compartir posición.
5. Después de vergüenza → pedir revancha.
6. Después de poder bloqueado → explicar cómo desbloquearlo.
7. Después de llegar al 80% del cupo → mostrar upgrade.

## Checklist UX para Codex

Cada pantalla debe cumplir:

- Acción principal clara.
- CTA visible above the fold en mobile.
- Copy emocional, no técnico.
- Elemento de comunidad.
- Elemento de progreso o estado.
- Elemento compartible.
- No lenguaje de apuestas.
- Diseño consistente con Stadium Social.

## Conclusión

La UI actual es una buena base visual, pero el frontend final debe convertirse en una máquina de loops sociales. El usuario no debe sentir que está llenando una quiniela. Debe sentir que está jugando reputación, orgullo, humor y pertenencia contra su comunidad.
